Statement vs. Expression

Erster Teil der „So verstanden" Serie: Der Versuch mittels einem Blogpost ein tieferes Verständnis für Konzepte zu entwickeln.
Sinn und Zweck ist es hierbei alle Logikmuster darzustellen und verständlich zu erklären.


Motivation

Im Zuge des Informatikstudiums wird man Autor vieler komplexer Softwaresystemen. Dieser Prozess, ganz besonders falls es schon einen Auftraggeber gibt oder falls das System in Kollaboration mit anderen entwickelt wird, erfordert "gute" Kommunikation.

(Einer der vielen) Eigenschaften von guter Kommunikation:

  1. Einfachheit (Gegenteil: Kompliziert)
  2. Ordnung (Gegenteil: Unübersichtlichkeit)
  3. Prägnanz (Gegenteil: Weitschweifigkeit)

Wie kommuniziert man nun prägnant? Nun, es erfordert präzise, treffende Wörter.

„Wir müssen den Throughput unseres Tools maximieren. Das Processing sollte separated und einzelne Parts parallel processed werden.“

Was gemeint ist: Strukturiere Arbeit wie ein Fließband.

„Deren Schnittstelle passt nicht auf unsere. Kannst du eine weitere Schnittstelle zur Konversion zwischen unseren beiden schalten?“

Was gemeint ist: Benutze einen Adapter.

Schlussfolgerung: Es gilt die richtigen Wörter zu finden. Damit einem das gelingt befasst man sich mit Definitionen und Beziehungen ähnlicher Wörter.

Statement vs. Expression

console.log("Hello World");

Wie würde man so etwas bezeichnen? Eine Zeile Code?

AssetLoader.mathFont2.draw(batch, world.getEnemy()[i].getEquation().getQuestion(), world.getEnemy()[i].getCircle().x-20, 
          world.getEnemy()[i].getCircle().y-10);

Oh Mist, ein Zeilenumbruch: Eine Zeile reicht nicht - ist es jetzt noch eine Zeile Code? Zu unpräzise!

Lass Klartext sprechen, hier sind meine Definition:

Expression

Ein expression (dt.: Ausdruck) ist eine Sequenz von Variablen und Operatoren, die entweder zu einem Wert evaluieren oder eine Wirkung erzielen.

Operatoren? Die hier:

+  -  *  /  %  =  ++  --  <  >  <=  >=  ==  !=  =  +=  *=  ||  &&  |  ^  &  ?  : /=  %=  << >>  >>> (u.v.m)

Beispiele für Ausdrücke:

  1.   42, "Str", true Gibt 42, "Str" oder true zurück. Wird auch Literal genannt.
  2.   3 + 1Gibt das Ergebnis von 3 + 1 zurück.
  3.   "1. " + array[0]Evaluation wo zum Literal "1. " der Inhalt aus array[0] konkateniert wird. Danach gibt es diesen Wert zurück.
  4.   x_coord > y_coordEvaluiert ob x_coord größer als y_coord ist. Gibt entweder true oder false zurück.
  5.   pi = 3.141Als Wirkung wird pi den Wert 3.141 zugewiesen, danach gibt es den Wert von pi zurück.
  6.   System.out.println(42)Als Wirkung ruft es die Methode toString() auf. Der Rückgabewert der Methode wird zurückgegeben.

Der im Deutschen verwendete Begriff "Ausdruck" darf nicht mit dem mathematischem "Ausdruck" verwechselt werden. So besteht ein Ausdruck in der Mathematik aus den Termen. In der Programmierung sind jedoch die Ausdrücke schon der noch-syntaktisch-korrekte, kleinste Bestandteil eines Programms.

Statement

Programme bestehen in ihrer elementarsten Form aus einzelnen Instruktionen, besser bekannt als "statements" (dt.: Anweisungen).
Die einfachsten Arten von Ausdrücken:
  1. Deklarationdouble pi;
  2. Zuweisungpi = 3.141; ((✷) War das nicht ein Ausdruck?)
  3. Ausführungsreihenfolgereturn;, break;, continue;
  4. Funktionenaufrufrand();
  5. Das leere Statement;

Da fehlt aber noch einiges: Compounding

"Compounding" - das Zusammensetzen von Expressions und Statements - macht die Sprache erst wirklich mächtig.

Compound Expressions

Vorhin schon gezeigt: So wie mathematische Terme sich zusammensetzen lassen, sofern sie vom gleichen Objekttyp (Zahlen, Mengen, Körper, ...) sind gilt das gleiche auch in der Programmierung bei gleichem Datentyp. Seien e, r als Integer-Variablen deklariert:

2 * 3 * 8 → 2 * r * ((2+2) * 2) → 2 * ++r * ((2+2) * rand()) → (e = 2) * ++r * ((2+2) * rand())

Compound Statements

Von einem compound statement spricht man sobald statements in geschweiften Blöcken { } gefasst sind

        {    
          System.out.println("Hello World");
          int pi = 3;
          pi++;
        }

Dieser eine Block wird nun als ein einziger statement gefasst, obwohl er mehrere statements enthält

Mit diesem Wissen kann man die übrigen Typen von statements ewähnen die - obwohl sie nach Spezifikation nur ein einzelnes statement akzeptieren - meistens mit mehreren statements in (einem Block zusammengefasst) kombiniert werden:

  1. if-Bedingungif (input == null) [statement]
  2. switch-Anweisungswitch (input) [statement]
  3. Schleifen (for, while, do while)while (i > 0) [statement]
  4. Methodendeklarationpublic int rand() [block]
  5. Klassendeklarationpublic class Auto [block]
  6. Ausnahmen behandelntry [block] catch [block] finally [block]
  7. Ausnahmen werfenthrow [expression]
  8. Kritischer Abschnittsynchronized (this) [block]
Mit [block] bestehend aus einem/mehreren [statement] und ein [statement] bestehend aus einem/mehreren [expresion].

Expression Statements (✷)

Es gibt wie man gesehen hat expressions die auch für sich alleine stehen können; es sind also praktisch schon statements.
  1. Deklaration + Zuweisungint pi = 3.141;
  2. Inkrement / Dekrementi++;, --j;
  3. Methodenaufrufrand();
  4. Klasseninstanzerzeugungnew Object();

Man erkennt expression statements müssen auf ein Semikolon enden.
